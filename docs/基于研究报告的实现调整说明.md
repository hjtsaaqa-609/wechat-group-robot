# 基于研究报告的实现调整说明

本文档说明项目根据《企业微信群（内部群与外部群）定期群发消息的设计与实现研究报告》已完成的代码调整。

## 已落地的调整

### 1. 消息通道从单一 Webhook 升级为可路由结构

原实现只有“企业微信群机器人 Webhook + markdown”单一通道。

当前已扩展为三类通道：

- `internal-webhook`
- `internal-app-message`
- `external-group-task`

对应研究报告中的“内部群与外部群分通道治理”建议。

### 2. 内部群支持应用消息能力

新增了企业微信自建应用消息通道，支持：

- `markdown`
- 超长内容自动降级为文件附件
- `enable_duplicate_check`
- `duplicate_check_interval`

这对应研究报告中“内部群优先使用 message/send，并结合重复消息检查”的建议。

### 3. 外部群不再按静默直发思路设计

新增的 `external-group-task` 通道不是直接发到客户群，而是调用企业微信“创建群发任务”接口：

- `/cgi-bin/externalcontact/add_msg_template`

这样更符合研究报告中“外部群必须经过成员确认执行”的合规约束。

### 4. 定时任务增加抖动机制

新增 `jitterSeconds`，调度时会在触发点后随机延迟执行，避免所有任务集中在整点触发。

这对应研究报告中“避开每小时 0 分或 30 分拥塞窗口”的建议。

### 5. 增加运行中防重入保护

同名任务如果上一轮尚未完成，新的调度触发会直接跳过，避免并发重复发送。

### 6. 超长 markdown 自动降级

研究报告要求企业微信 markdown 控制在 2048 字节内。当前实现已支持：

- 若在 2048 字节内，直接发送 markdown
- 若超过阈值，自动上传 `.md` 附件并发送文件消息

### 7. 修正周报状态持久化口径

周报持久化时的“平台机器人数量”已统一为：

- `trialRun + formalOperation`

不再使用全量机器人数量，保证与周报展示口径一致。

### 8. 文档统一迁移到 docs

项目说明文档已经统一放到 `docs/` 下维护，根目录 `README.md` 仅保留入口说明。

## 当前未完全展开的部分

以下能力已按研究报告方向预留，但未在本轮做成完整业务闭环：

- 外部群发结果回收：`get_groupmsg_send_result`
- 外部群发提醒成员执行：`remind_groupmsg_send`
- 模板卡片交互：`template_card`、`update_template_card`
- 在线文档承载：`wedoc/create_doc`、`wedoc/doc_share`
- 外部群脱敏摘要与门户链接联动

这些能力后续可以在现有通道模型上继续追加，不需要再推翻当前架构。
