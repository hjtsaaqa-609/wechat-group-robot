# 部署与使用说明

## 项目概述

本项目负责从 `http://das.i-pv.cn` 拉取 DAS 数据，生成经营周报，并按配置路由到企业微信消息通道。

当前默认内置报表：

- `weekly-operations`：按自然周汇总的经营周报（整体经营 + 各部门 KPI）

当前支持的企业微信通道：

- `internal-webhook`：内部固定群机器人 Webhook
- `internal-app-message`：企业微信自建应用消息
- `external-group-task`：企业微信外部群发任务创建接口

## 安装依赖

```bash
npm install
```

## 环境变量

在项目根目录创建 `.env`，参考 [`.env.example`](/Users/michael/Documents/AI%20Codex/Wechat%20group%20chat%20robot/.env.example)：

```bash
DAS_BASE_URL=http://das.i-pv.cn
DAS_USERNAME=your_das_username
DAS_PASSWORD=your_das_password

# 内部固定群 - 机器人 Webhook
WECOM_WEBHOOK=https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=your_key

# 内部应用消息 / 外部群发任务 - 企业微信应用凭证
WECOM_CORP_ID=your_wecom_corp_id
WECOM_CORP_SECRET=your_wecom_corp_secret
WECOM_AGENT_ID=1000002
```

## 任务配置

编辑 [`config/jobs.json`](/Users/michael/Documents/AI%20Codex/Wechat%20group%20chat%20robot/config/jobs.json)。

核心字段：

- `cron`：定时表达式
- `jitterSeconds`：触发抖动秒数，用于避开整点集中发送
- `channel`：企业微信发送通道
- `reports`：要发送的报表列表

默认示例：

```json
{
  "name": "DAS周报",
  "enabled": true,
  "cron": "5 8 * * 1",
  "jitterSeconds": 0,
  "channel": {
    "type": "internal-webhook",
    "webhookEnv": "WECOM_WEBHOOK",
    "maxMarkdownBytes": 2048
  },
  "reports": [
    {
      "type": "weekly-operations",
      "detailLimit": 5
    }
  ]
}
```

```json
{
  "channel": {

  }
```

### 内部应用消息

```json
{
  "channel": {
    "type": "internal-app-message",
    "corpIdEnv": "WECOM_CORP_ID",
    "corpSecretEnv": "WECOM_CORP_SECRET",
    "agentIdEnv": "WECOM_AGENT_ID",
    "touser": "@all",
    "duplicateCheckInterval": 1800
  }
}
```

### 外部群发任务

```json
{
  "channel": {
    "type": "external-group-task",
    "corpIdEnv": "WECOM_CORP_ID",
    "corpSecretEnv": "WECOM_CORP_SECRET",
    "sender": "zhangsan",
    "chatType": "group",
    "textPrefix": "【客户群摘要】"
  }
}
```

说明：

- `external-group-task` 走的是“创建群发任务”链路，不是静默直接发到客户群。
- 任务创建后仍需企业微信侧成员确认执行，符合研究报告中的外部群合规要求。

## 推荐的定时触发方式

如果只有少量固定任务，推荐直接使用操作系统自己的调度器调用单次执行。

### Linux / macOS

推荐使用系统 `cron`。

先确认脚本可执行：

```bash
chmod +x scripts/run-job.sh
```

手动测试：

```bash
./scripts/run-job.sh "DAS周报"
```

Linux `crontab` 示例：

```bash
5 8 * * 1 /bin/bash /Users/michael/Documents/AI\ Codex/Wechat\ group\ chat\ robot/scripts/run-job.sh "DAS周报" >> /Users/michael/Documents/AI\ Codex/Wechat\ group\ chat\ robot/logs/weekly-report.log 2>&1
```

### Windows 10 / Windows 11

推荐使用“任务计划程序”调用：

[`scripts/run-job.ps1`](/Users/michael/Documents/AI%20Codex/Wechat%20group%20chat%20robot/scripts/run-job.ps1)

详细步骤见：

[Windows 10 利旧部署说明](/Users/michael/Documents/AI%20Codex/Wechat%20group%20chat%20robot/docs/Windows%2010%20利旧部署说明.md)

### GitHub Actions

如果代码已经托管到 GitHub，且希望完全摆脱本地机器常驻运行，可以让外部定时器触发 GitHub Actions 单次执行。

当前 GitHub Actions workflow 不再配置内置 `schedule`，避免仓库内部自动触发。

详细步骤见：

[GitHub Actions 定时部署说明](./GitHub%20Actions%20定时部署说明.md)

## 当前实现口径

- 程序会先调用 `/api/auth/login` 校验 DAS 账号密码，再抓取报表数据。
- 当前 `DAS周报` 默认输出“整体经营 + 各部门 KPI”模板。
- 周报中的“机器人累计上线”使用 `试运行 + 正式运营` 口径。
- 客户满意度部分当前仅展示“年度累计投诉数量”。
- 研发部当前只展示已上线的 3 个 `每 GW` 指标：远程接管、现场维护、毛刷更换。
- 项目部时效目标当前使用原运营模板里的 4 个 backlog/持续时长指标做代理展示。
- 周报按 DAS 当前实时接口自动生成，统计周期默认取最近一个已结束的周六至周五周期。
- `config/jobs.json` 中的默认发送时间只在运行 `npm run start` 的内部定时模式下生效；GitHub Actions 由外部定时器触发。
- 内部通道的 markdown 超过 2048 字节时，会自动降级为多条文本消息发送。
- 旧版运营周报模板已保留代码备份在 [src/reports/weekly-operations-archive.ts](/Users/michael/Documents/AI%20Codex/Wechat%20group%20chat%20robot/src/reports/weekly-operations-archive.ts)。
